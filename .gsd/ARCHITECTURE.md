## Overview

**Kino** is a mood-driven movie discovery app. Users describe how they *feel* in natural language, and an AI engine translates that into curated movie recommendations. The system is composed of a **FastAPI Python backend** and a **Flutter mobile frontend**, with no database — all data is fetched live from TMDB and Gemini AI at request time.

## Data Flow

### Standard Request (e.g., Trending)
1. Screen calls `ApiService.getTrending()`
2. `http.get('http://127.0.0.1:8000/api/v1/movies/trending')`
3. `routes.py` → `TMDBService.fetch_trending()`
4. `httpx.get('https://api.themoviedb.org/3/trending/movie/week')`
5. `_normalize()` applied to each result
6. JSON → Flutter `json.decode()` → `setState()` → rebuild

### Vibe Search (AI Pipeline)
1. User types natural language → `ApiService.searchVibe(query, sort)`
2. `routes.py` → `AIService.analyze_vibe(query)` → Gemini 2.0 Flash
3. Gemini returns `{mood, genres, tone, rating, keywords}`
4. `TMDBService.discover_movies()` called × 2 (EN + HI) with Gemini filters, concurrently
5. Top 30 results → enhancement pass (runtime, trailer, certification) concurrently
6. Full response `{movies[], metadata{}}` → Flutter
7. `metadata` displayed as colored tag chips in `VibeCheckScreen`

## Conventions

**Naming (Flutter):** `snake_case` for files, `PascalCase` for classes, `camelCase` for methods/variables

**Structure:** Feature-by-screen organization in Flutter; service-layer pattern in backend  
**Error handling:** Try/catch in Flutter screens with `setState`; try/except in backend services with fallback returns  
**Testing:** `flutter_test` dev dependency present; `test.py` / `test_ai.py` scripts in backend (not a test suite)  
**No formal test suite exists in either codebase**
